# Reflection Workflow Test

> Tests the reflection workflow's ability to scan conversations, extract patterns
> via the reflector agent, and aggregate them into REFLECTIONS.md

## Prerequisites

- Mastra server running on port 4111
- Reflection enabled in `agent.toml [reflection]`
- Memory storage accessible with conversation threads
- Reflector agent model configured in `agent.toml [models]`

## Test Setup

```bash
# Ensure reflection is enabled
grep -A 7 "\[reflection\]" .agent/agent.toml

# Expected output:
# [reflection]
# enabled = true
# cron_schedule = "*/5 * * * *"
# max_messages_per_run = 1000
# min_batch_size = 3
# max_pending_minutes = 30
# reflections_md_path = "REFLECTIONS.md"
# reflection_state_path = "reflection-state.json"
```

## Test Scenario: Extract Patterns from Conversations

### Step 1: Ensure Conversations Exist

**Action:** Verify conversation threads exist in memory storage

The workflow scans memory threads for user/assistant exchange pairs. At least one thread should have multiple exchanges.

### Step 2: Clear State (Optional)

**Action:** Reset processed message tracking

```bash
# Remove state file to force full rescan
rm -f .agent/reflection-state.json
```

### Step 3: Trigger Reflection Workflow

**Manual Trigger:**
```bash
curl -X POST http://localhost:4111/api/workflows/reflection/start-async \
  -H "Content-Type: application/json" \
  -d '{"inputData": {"resourceId": "interactive-agent"}}'
```

### Step 4: Verify REFLECTIONS.md Output

**Expected File:** `.agent/REFLECTIONS.md`

**Expected Structure:**
```markdown
# Reflections

> Auto-generated by reflection workflow. Last updated: <timestamp>
> Scanned: N threads, N messages, N raw patterns

## Patterns

### Attention Signals
- <description> (confidence: N.NN, occurrences: N, evidence: <text>)

### Decision Markers
- <description> (confidence: N.NN, occurrences: N, evidence: <text>)

### Noise Patterns
- <description> (confidence: N.NN, occurrences: N, evidence: <text>)

## Heuristics
- **heuristic_N**: <condition> → <action> (weight: N.N)
```

### Step 5: Test Pattern Aggregation (Second Run)

**Action:** Trigger the workflow again after more conversations

**Expected:** New patterns merge with existing ones:
- Similar patterns: occurrences increment, confidence updated
- New patterns: appended to existing
- Duplicate patterns: deduplicated by Jaccard similarity

```markdown
### Attention Signals
- <merged pattern> (confidence: 0.93, occurrences: 3, evidence: ...)
```

### Step 6: Test Existing-Pattern Context

**Action:** Check workflow logs during second run

**Expected Logs:**
```
[Reflection Workflow] Loaded N existing patterns, N heuristics as context
[Reflection Workflow] Analyzing N pairs in N batches
```

This confirms the reflector agent receives existing patterns, enabling it to skip known observations and focus on new findings.

### Step 7: Test Incremental Processing

**Action:** Check state file after run

```bash
cat .agent/reflection-state.json | python3 -m json.tool
```

**Expected:**
```json
{
  "lastRunAt": "<recent timestamp>",
  "processedMessageIds": ["msg-1", "msg-2", "..."]
}
```

**Action:** Trigger another run without new conversations

**Expected:** "No pairs to analyze, skipping" — only unprocessed messages are scanned

## Test Scenario: Max Limits

### Step 1: Generate Many Patterns

**Action:** Create 150+ unique patterns across many runs

**Expected:** Only top 100 kept (sorted by confidence * occurrences)

### Step 2: Generate Many Heuristics

**Action:** Create 30+ unique curation suggestions across runs

**Expected:** Only top 20 heuristics kept

## Test Scenario: buildBatchPrompt Unit Tests

Unit tests in `tests/services/reflection-service.test.ts` cover:

| Test | Description |
|------|-------------|
| Without existing patterns | Basic prompt with exchanges, no "Already Known" sections |
| With patterns + heuristics | Both context sections present, evidence excluded, "Focus on genuinely NEW" |
| With empty arrays | Same as no existing patterns |
| With patterns only | Patterns section present, heuristics section absent |

```bash
bun test tests/services/reflection-service.test.ts
```

## Cleanup

```bash
# Remove generated files
rm -f .agent/REFLECTIONS.md
rm -f .agent/reflection-state.json
```

## Failure Modes

| Issue | Diagnostic |
|-------|------------|
| No pairs collected | Check memory threads exist, verify resourceId, check processedMessageIds |
| Reflector agent fails | Check model config in agent.toml, check API key, review error logs |
| Patterns not merging | Check Jaccard similarity threshold (0.6), verify same pattern type |
| Full rescan each time | Check reflection-state.json exists and is writable |
| Known patterns re-discovered | Check existing-pattern context loaded (see logs) |

## Validation Queries

```bash
# Check if patterns were written

# Check heuristics count

# Check last run timestamp

# Check state file

```
